## 核心特性

既能享受低代码10分钟配置出用户/机构管理等基础模块的高效，又能凭借优雅的扩展机制，轻松应对实时数仓、复杂工作流、数据权限等严峻的企业级挑战。

| **特性维度**    | **传统Java低代码框架**                                        | **OpenJee 开源平台**                                        |
| :---------- | :----------------------------------------------------- | :------------------------------------------------------ |
| **后端架构**    | 代码生成器模式：每张表生成独立Entity、Controller、Service、Mapper，代码量庞大。 | 统一接口 + 钩子扩展：单一实体、Controller、Service、Dao处理所有表的CRUD，结构极简。 |
| **前端界面**    | 生成代码可能复杂或定制性差，或动态渲染UI僵化。                               | 生成可读性高的Vue3代码：保留代码生成优势，便于深度定制和优雅扩展。                     |
| **扩展方式**    | 修改生成代码或使用有限插件，复杂需求下低代码优势减弱。                            | Around（环绕）钩子：通过实现特定钩子接口，以AOP方式非侵入式注入复杂业务逻辑，保持核心简洁。      |
| **透明度与可控性** | 生成代码可见，但底层引擎可能封闭。                                      |                                                         |
